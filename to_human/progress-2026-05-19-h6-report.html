Calibration-Guided Precision Assignment

Progress report on whether short pre-training precision checks can identify precision-sensitive modules and guide stable low-precision LoRA fine-tuning.

Report date: 2026-05-19 Project status: active Main hypothesis: H6 Hardware focus: RTX 3090

Executive Takeaway

The project now has two defensible findings. First, cheap calibration signals identify safe and unsafe module-level precision candidates in Qwen2.5-0.5B LoRA: high-risk MLP down projections show large perturbation loss deltas, while selected late-layer gate/up projections remain quality-safe across three seeds. Second, generic QLoRA 4-bit NF4 becomes useful at Qwen2.5-7B as a memory-capacity trade-off: it saves 23.32% peak memory across three 500-step seeds while staying inside the locked 1% eval-loss gate, but it is about 20% slower than bf16.

The current claim is not a speed win. The clean claim is: calibration predicts precision sensitivity at small scale, and hardware-backed QLoRA provides a robust 7B memory-capacity trade-off.

Research Question

Before LoRA fine-tuning, can a short precision check identify which model modules are precision-sensitive, so a frozen mixed-precision policy can match bf16 validation quality while saving resources or expanding the stable fine-tuning envelope?

The original static-island idea, keeping norms in fp32, became a weak anchor rather than the main result. Qwen2 RMSNorm already upcasts its reduction arithmetic internally, so hand-picking norm islands does not produce a meaningful effect. The stronger path is measured precision sensitivity.

Finding 2: 7B QLoRA Is a Memory-Capacity Trade-Off

Current Interpretation

What is supported

  • Module sensitivity is measurable before training.
  • Outlier-based calibration predicts MLP projection perturbation sensitivity.
  • Frozen fake-int8 policies selected by calibration preserve quality across seeds.
  • At 7B, QLoRA provides repeatable memory savings within the 1% quality gate.

What is not supported

  • fp32 norm islands are not a strong intervention for Qwen2.5 LoRA.
  • Generic bitsandbytes gives no resource win at 0.5B or 1.5B in this setup.
  • QLoRA is not a throughput improvement on the RTX 3090.
  • The current 7B QLoRA result is not yet calibration-guided selective precision.

Recommended Next Step

Stop repeating generic bitsandbytes baselines for now. The result is clear enough. The next experiment should test whether the calibration-guided sensitivity story transfers from 0.5B to 7B.

  1. Run 7B calibration on a smaller module panel first, because all-module calibration may be expensive.
  2. Include known 0.5B high-risk analogs, late-layer gate/up candidates, and a few attention controls.
  3. Compare whether 7B sensitivity rankings agree with 0.5B, then decide whether selective low precision at 7B is plausible.
If the 7B calibration ranking transfers, the paper story becomes stronger: cheap probes identify precision-tolerant modules, and the hardware-backed 7B result shows why memory capacity matters.
