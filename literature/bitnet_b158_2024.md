# The Era of 1-bit LLMs: All Large Language Models are in 1.58 Bits

- **Authors:** Shuming Ma, Hongyu Wang, Lingxiao Ma, Lei Wang, Wenhui Wang, Shaohan Huang, Li Dong, Ruiping Wang, Jilong Xue, Furu Wei
- **Year:** 2024
- **Source:** https://arxiv.org/abs/2402.17764
- **DOI:** https://doi.org/10.48550/arXiv.2402.17764

## Summary

BitNet b1.58 proposes a native ternary-weight LLM architecture where weights take values in {-1, 0, 1}. It argues that low-bit LLMs can be trained from scratch to match FP16/BF16 Transformer baselines at sufficient scale while reducing memory, latency, and energy.

This is not the main path for the current project because it changes the architecture/training recipe rather than assigning precision within a standard LoRA fine-tuning run. It is included as boundary context for the INT4/subbyte landscape.

## Relevance To This Project

BitNet shows that the long-term endpoint of low-precision LLM work may be precision-native architectures, not just quantized versions of full-precision Transformers. For this project, it mainly motivates caution: results from native low-bit architectures may not transfer to post-hoc adaptive precision islands in ordinary models.

## Key Takeaways

- Native low-bit training can differ fundamentally from post-training quantization.
- Low precision may require architecture-level design, not only dtype assignment.
- FP16/BF16 baselines remain the comparison point for quality and scaling claims.

## Evidence Gaps

- It is not a LoRA, BF16/FP32 precision-island, or adaptive assignment paper.
- It does not provide per-operation stability signals for standard Transformer fine-tuning.
- It should not be treated as evidence that arbitrary INT4/INT1 demotion is safe in existing LLMs.
