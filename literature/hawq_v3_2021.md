# HAWQ-V3: Dyadic Neural Network Quantization

- **Authors:** Zhewei Yao, Zhen Dong, Zhangcheng Zheng, Amir Gholami, Jiali Yu, Eric Tan, Leyuan Wang, Qijing Huang, Yida Wang, Michael W. Mahoney, Kurt Keutzer
- **Year:** 2021 / ICML 2021
- **Source:** https://arxiv.org/abs/2011.10680
- **Proceedings:** https://proceedings.mlr.press/v139/yao21a.html

## Summary

HAWQ-V3 extends Hessian-aware mixed-precision quantization into an integer-only deployment framework. It formulates precision assignment as an integer linear programming problem that balances model perturbation against constraints such as memory footprint and latency. It reports mixed INT4/INT8 deployment results and emphasizes that avoiding floating-point conversion overhead matters for real speedups.

The most relevant contribution is not the specific CNN result, but the framing: precision assignment is a constrained optimization problem over quality loss and system cost.

## Relevance To This Project

H6 can adopt the same structure at small scale: collect sensitivity/stability signals, define a cost model for memory and throughput, then freeze a policy before final evaluation. HAWQ-V3 also warns that fake low precision without useful kernels may not improve wall-clock time on the available GPU.

## Key Takeaways

- Mixed precision should optimize quality and hardware constraints jointly.
- INT4 can be useful in some paths, but conversion overhead and kernel support determine real benefit.
- A policy trace is important: the system should explain why each layer received a precision.

## Evidence Gaps

- The paper is not LLM-specific and focuses on inference deployment.
- It uses Hessian/ILP machinery that may be heavier than a LoRA pilot needs.
- It does not cover BF16/FP16/FP8 training stability signals.
