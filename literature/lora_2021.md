# LoRA: Low-Rank Adaptation of Large Language Models

- **Authors:** Edward J. Hu, Yelong Shen, Phillip Wallis, Zeyuan Allen-Zhu, Yuanzhi Li, Shean Wang, Lu Wang, Weizhu Chen
- **Year:** 2021
- **Source:** https://arxiv.org/abs/2106.09685
- **DOI:** https://doi.org/10.48550/arXiv.2106.09685

## Summary

LoRA freezes pretrained model weights and injects trainable low-rank matrices into Transformer layers. It reduces the number of trainable parameters dramatically and lowers GPU memory requirements while maintaining comparable quality to full fine-tuning in several settings.

The paper reports that, compared with GPT-3 175B fine-tuning with Adam, LoRA can reduce trainable parameters by 10,000x and GPU memory by 3x. It also avoids extra inference latency compared with some adapter methods.

## Relevance To This Project

LoRA defines the adaptation regime used in H1. Because the base model is frozen and only adapters are trained, mixed-precision effects may differ from full fine-tuning or pretraining. That makes LoRA a feasible but distinct testbed for precision-island hypotheses.

## Key Takeaways

- LoRA makes limited-GPU fine-tuning scientifically and practically feasible.
- The base model remains frozen, so instability may concentrate in activations, adapter gradients, optimizer states, or loss computation.
- A LoRA pilot should not be overgeneralized to full-parameter fine-tuning without follow-up experiments.

## Evidence Gaps For H1

- The paper does not study BF16/FP32 precision islands.
- It does not report dtype-sensitive stability metrics such as loss spikes or NaN/Inf incidence.
